# AI 보조 주식 스코어링/트레이딩 시스템 — 마스터플랜 v3

> 한 줄 요약: **AI가 구조를 짜고, 사람이 선을 긋는다.** 코드는 단계별로 짓되,
> 각 단계는 다음으로 넘어가기 전 통과해야 하는 **게이트**가 있다.
> 가장 중요한 게이트는 "비용 반영 백테스트에서 엣지가 보이는가"이며,
> 통과 전엔 실제 돈을 절대 넣지 않는다.

> **v2 → v3 무엇이 바뀌었나**
> 1. **변동성 중복 제거.** v2는 변동성을 점수에서 두 번(위험조정 모멘텀 + 저변동성)
>    + 비중에서 한 번 = 사실상 "저변동성 한 가지"에 쏠렸음. v3는 점수에서 변동성을
>    **한 번만**(위험조정 모멘텀) 쓰고, 진짜 분산은 결이 다른 팩터(밸류/퀄리티)로 미룸.
> 2. **테스트 단계 보수성 재정의.** 추세 게이트·보유 종목 수 같은 '전략 보수성'은
>    미리 깔지 말고 **백테스트에서 ON/OFF로 시험할 가설**로 둔다. 대신 '평가 보수성'
>    (비용 가정·의심·아웃오브샘플)은 최대로 빡세게.

---

## 0. 역할 분담 (대전제)

- **AI(Claude/Hermes)**: 구조 설계, 코드 작성, 기본값 제안, 디버깅, 함정 차단.
- **사람(운영자)**: 팩터/임계값/비중 방식/리스크 한도 **결정**, 게이트 통과 **판단**.
- AI는 "이 전략이 통하는가"에 답하지 못한다. 그건 **검증**으로만 나오고,
  검증에는 압축 불가능한 **시간**이 든다.

---

## 1. 핵심 원칙

1. **검증 우선.** 만드는 시간(AI가 당겨줌) ≠ 검증 시간(절대 안 당겨짐).
2. **페이퍼 먼저, 실거래는 한참 뒤.**
3. **단순함이 실력.** 손잡이가 많을수록 과거에 깎아 맞추게 됨(과최적화).
   "뭘 안 넣을지 아는 것"이 고급 기술.
4. **조용한 실패 경계.** 작게 만들고 출력을 눈으로 확인.
5. **시장 비종속 구조.** 데이터/브로커 어댑터 분리 → US↔KR 재사용.

---

## 2. "보수성"의 두 종류 (v3 핵심 개념)

| | 전략 보수성 | 평가 보수성 |
|---|---|---|
| 예 | 추세 게이트, 보유 종목 수, 쏠림 제한 | 비용 가정, 결과 의심, 아웃오브샘플 |
| 테스트 단계 태도 | **미리 깔지 말고 ON/OFF 가설로 시험** | **최대로 빡세게** |
| 이유 | 보수 버전이 나을지 공격 버전이 나을지는 백테스트가 답할 문제. 미리 막으면 엣지가 게이트 밖에 있어도 못 봄 | 비관적으로 가정할수록 실전에서 덜 깨짐 (자기를 안 속이는 보수성) |

→ 보수성은 결과 보고 **고르는 것**이지 미리 **깔고 가는 것**이 아니다.
   단, 백테스트로 버전을 정한 뒤 페이퍼 단계에선 **정해진 버전 그대로** 돌린다.

## 3. 시스템 아키텍처

```
[데이터 어댑터]  →  [멀티팩터 스코어러]  →  [백테스터(정직)]   (검증 트랙)
  yfinance          섹터중립 z-score        비용·룩어헤드·생존편향 처리
  + 섹터(위키 GICS)        │                  + 게이트 ON/OFF 비교
                          ▼
                 [실행기 + 로거]  ←  [스케줄러]                 (운영 트랙)
                  Alpaca 페이퍼       정해진 시간 자동 실행
                  역변동성 비중
```

| 모듈 | 역할 | 기술 | 비고 |
|---|---|---|---|
| 데이터 어댑터 | 가격·유니버스·섹터 | yfinance + 위키 GICS | KR이면 pykrx |
| 스코어러 | 멀티팩터 점수 | pandas | 시장 비종속 |
| 백테스터 | 과거 성과(정직) | pandas | **엣지 판별 게이트** |
| 실행기/로거 | 모의 주문 + 기록 | Alpaca 페이퍼(무료) | 키 필요 |
| 스케줄러 | 자동 실행 | cron / VPS | 전역 후 |

## 4. 스코어러 설계 (v3)

### 4.1 점수 팩터 — 변동성은 여기서 한 번만

**핵심 신호 (가격 기반, 지금 가능):**
- **위험조정 모멘텀** = 모멘텀(12-1) ÷ 변동성.
  *흔들림 대비* 잘 오른 종목 선호. 변동성을 점수에 반영하는 건 **이거 하나로 끝**.
- (대안 비교용) **순수 모멘텀(12-1)** — 위험조정 버전과 *둘 중 뭐가 나은지
  백테스트로 비교*. 둘을 **쌓지 않는다**(둘 다 모멘텀 계열이라 쌓으면 중복).

**진짜 분산 팩터 (Phase 1.5, 데이터 게이트):**
- **밸류(PER/PBR)·퀄리티(ROE 등)** — 모멘텀과 결이 다른 신호라 진짜 분산이 됨.
  단 yfinance 재무는 부실 → **신뢰 가능한 재무 데이터 확보 후** 추가.
  > 솔직히: 무료 가격 데이터만으론 견고한 신호가 사실상 '모멘텀' 한 계열뿐.
  > 그래서 v3 초기 점수는 위험조정 모멘텀 중심으로 단순하게 가고,
  > 분산은 펀더멘털 데이터가 생길 때 진짜로 추가한다.

### 4.2 섹터 중립화
- 각 팩터 z-score를 **섹터 안에서** 계산(섹터 평균 차감) → "핫한 한 섹터 몰빵" 방지.
- 섹터: S&P500 위키 표의 **GICS Sector** 컬럼(무료).
- 가드: 섹터당 종목이 너무 적으면 비교 불안정 → **섹터당 최소 개수 가드** 둘 것.

### 4.3 점수 합산
```
combined_score = Σ ( wᵢ × zscore_sector(factorᵢ) )
```
- 가중치는 운영자 결정(기본: 동일가중). 잘게 돌리지 말 것(과최적화).

### 4.4 추세 게이트 — 가설로 취급
- 200일 MA 위만 통과시키는 옵션. **기본값을 ON으로 깔지 않는다.**
  백테스트에서 **ON/OFF 둘 다 돌려** 어느 쪽이 나은지 데이터로 결정.

### 4.5 포지션 비중 — 변동성의 '다른 역할'
- **역변동성 비중**(1/변동성, 합 1 정규화). 덜 흔들리는 종목에 더 싣기.
  이건 점수가 아니라 **리스크 관리** 목적이라 4.1의 변동성과 역할이 달라 중복 아님.

## 5. 단계별 로드맵 (게이트 포함)

### Phase 0 — 준비
- [ ] Alpaca 페이퍼 계좌(무료) + 키
- [ ] `pip install yfinance pandas lxml alpaca-py`
- **게이트:** 페이퍼 잔고 조회 성공.

### Phase 1 — 스코어러
- [x] 단일 모멘텀 버전 완료
- [ ] 위험조정 모멘텀으로 전환 + 섹터 중립 z-score + 역변동성 비중
- **게이트:** 랭킹·비중이 납득 가능, CSV 저장.

### Phase 1.5 — (선택) 펀더멘털 팩터
- [ ] 신뢰 가능한 재무 데이터 확보 → 밸류·퀄리티 추가(진짜 분산)
- **게이트:** 결측·이상치 처리 후 점수 안정.

### Phase 2 — 정직한 백테스트  ★생명선★
- [ ] 월 1회 리밸런싱 시뮬(상위 N, 역변동성 비중)
- [ ] **전략 보수성은 가설로 비교:** 게이트 ON/OFF, 보유 종목 수 2~3개 버전
- [ ] **평가 보수성은 최대:**
      - 룩어헤드 차단(신호=리밸일까지 데이터, 체결=다음날 시가)
      - 비용 비관적 가정(슬리피지 편당 5~10bp + 회전율)
      - 생존편향 인지(현재 구성종목 → 결과가 실제보다 좋게 나옴, 한계 명시)
      - **아웃오브샘플 확인**(고른 버전을 '안 본 기간'에서 재검증)
- [ ] 지표: CAGR·변동성·샤프·MDD·**SPY 대비 초과수익**·회전율
- **과최적화 가드:** 버전은 2~3개만, 1등은 반드시 아웃오브샘플 재확인,
  파라미터 미세조정 금지.
- **게이트:** 비용 반영 + 아웃오브샘플에서도 SPY 대비 의미 있는 초과수익 + 견딜 MDD.
  → 통과 못 하면 멈추고 재설계. (못 이기면 그냥 인덱스가 답일 수 있음)

### Phase 3 — 페이퍼 트레이딩 + 로깅 (포워드 시계 ON)
- [ ] 스코어러 → Alpaca 페이퍼 주문, **백테스트로 정한 버전 그대로**
- [ ] 리밸런싱마다 점수·체결·잔고 로깅, VPS 스케줄 실행
- **게이트:** 수 개월(여러 국면) 페이퍼 성과가 백테스트와 크게 안 어긋남.
  ← 군 복무 기간과 겹쳐 돌리면 시간 절약.

### Phase 4 — 소액 실거래 (검증 완료 후만)
- [ ] 잃어도 되는 소액 + 손실 한도·**킬스위치** 먼저
- **게이트:** 모니터링·개입 가능한 환경(= 전역 후).

| 항목 | 기본값 | 의미 |
|---|---|---|

## 8. 조용한 실패 리스크 (점검)

- **과최적화:** 버전 최소화 + 아웃오브샘플 재확인 + 미세조정 금지.
- **룩어헤드:** 미래 정보 누수, 백테스트 1순위 점검.
- **생존편향:** 현재 구성종목만 쓰면 결과 과대평가, 한계 명시.
- **비용 누락:** 수수료·슬리피지·회전율 미반영 시 수익은 환상.
- **숨은 쏠림:** 같은 위험에 여러 번 베팅하지 않기(v3에서 변동성 중복 제거).
- **장 시간:** 미국장=한국 밤~새벽. 봇엔 무관, 직접 개입은 어려움.
- **킬스위치:** 실거래 전 자동 정지 조건 정의.
